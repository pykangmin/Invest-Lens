# 커밋 규칙

## 메시지 형식

Conventional Commits 형식을 따른다.

- 기능 추가: `feat(layer): 설명`
- 버그 수정: `fix(layer): 설명`
- 리팩터링: `refactor(layer): 설명`

## 스코프 (layer)

괄호 안에는 변경이 속한 영역을 명시한다:

- src/ 레이어명: `types`, `schema`, `data-loader`, `analysis`, `visualization`, `layout`, `shared`
- skills/ 파일명 (번호 prefix 제외): `assumptions`, `ingest`, `analyze`, `render`, `data-profile`, `data-analysis`, `insight`, `frontend-aesthetics`
- 상위 카테고리: `docs`, `infra`

## 커밋 타이밍 규칙

- 빌드가 깨진 상태에서는 커밋하지 않는다.
- 구현에 실패하면 `git revert`로 마지막 정상 상태로 복귀한 뒤 재시도한다.
- 반쯤 완성된 코드를 커밋하지 않는다.
