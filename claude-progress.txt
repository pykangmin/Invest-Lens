### 발견한 함정 / 정정

- **이전 메모 정정**: "node_modules 가 Linux/WSL 빌드본이라 전체 재설치 필요" → 사실 부분 누락. `@esbuild/win32-x64/` 디렉토리 하나만 비어있던 상태였고, `tsx`·`vite`·`@rollup/rollup-win32-x64-msvc` 는 모두 멀쩡. 향후 비슷한 진단 시 디렉토리 단위로 점검할 것.
- **postinstall ELF 트랩**: `node_modules/esbuild/bin/esbuild` 가 Linux ELF 로 남아있으면 npm rebuild 가 그것을 실행하려다 EBADF 로 실패. 해결: 그 파일 먼저 삭제 후 `npm install esbuild --force --include=optional`.
- **DB·시안 매핑 결과**: 시안 13개 슬롯 중 11개는 현 DB(503 종목 + 6 테이블) 로 즉시 채울 수 있음. 미해결은 (1) 시장 지수 카드(S&P/Dow 등의 가격 — global_environment 에 직접 없음), (2) 주요 이벤트 리스트의 데이터 소스(권장: DB 신호 변화 합성 — supertrend/macd/dominant_regime 전환).
- **API 라우트 로컬 실행**: 현재 `api/*.ts` 는 Vercel serverless 함수 형태. Vite dev 서버에서는 서빙 안 됨 → StockPage 의 `/api/company` 호출이 dev 환경에서는 404. `vercel dev` 또는 Vite 의 미들웨어 어댑터가 필요. 다음 세션 작업.
