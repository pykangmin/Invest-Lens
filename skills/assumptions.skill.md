## 2. 수익률 계산 방식 — 두 트랙 분리

| 트랙 | 방식 | 수식 | 적용 범위 |
|------|------|------|-----------|
| Internal | 로그 수익률 | $r_t = \ln(P_t / P_{t-1})$ | regime.py HMM 등 통계 모델 입력 |
| Display | 단순 수익률 | $R_t = (P_t - P_{t-1}) / P_{t-1}$ | 대시보드 표시, 스코어카드 계산 |

HMM은 시간 가산성·정규 분포 가정이 필요 → 로그 수익률. UI는 사용자 직관성 → 단순 수익률.

**Display Track 기간별 수익률 및 변동성**:

| 기간 | 코드 |
|------|------|
| 1주 (5거래일) | `(P_t - P_{t-5}) / P_{t-5}` |
| 1개월 (21거래일) | `(P_t - P_{t-21}) / P_{t-21}` |
| 3개월 (63거래일) | `(P_t - P_{t-63}) / P_{t-63}` |
| 1년 (252거래일) | `(P_t - P_{t-252}) / P_{t-252}` |
| 연환산 변동성 | `stdDev(R_d) × √252` |

> `src/data-loader/`에서 두 트랙을 별도 필드로 분리 저장. 절대 혼용 금지.

## 5. 이상치 처리 — 에셋 클래스별 분기

`asset_class === 'EQUITY'` → 주식 규칙 / `asset_class === 'COMMODITY'` → 원자재 규칙.

| 케이스 | EQUITY | COMMODITY |
|--------|--------|-----------|
| 음수 가격 | **drop** (물리적 불가) | **유효값 저장** + ⚠️ (2020-04-20 WTI -$37 사례) |
| 음수 거래량 | **drop** | **drop** |
| 일 수익률 이상치 | ±50% 초과 → ⚠️, 계산 포함 | ±80% 초과 → ⚠️, 계산 포함 |
| 음수 FCF | 유효값, 점수 0점 | — |
| PBR·PER 음수 | 유효값, Z-score 제외, 점수 0점 | — |
| 거시 Z-score > 4σ | ⚠️, 계산 포함 | ⚠️, 계산 포함 |
